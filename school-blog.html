March 25-29, 2024I was working at my job a lot this week, but luckily I was ahead of schedule for school because of everything I did last week. I mainly worked on the thread-safe queue assignment for my Operating Systems class. This was a pretty short assignment - I implemented a serial queue (backed by a linked list) and then just added locks to it to make it thread-safe. Locks were placed around anything that could cause a potential race condition (data that multiple threads are trying to access and change). I had to lock most of the enqueue and dequeue methods because changing the head and tail pointers could be a race condition. I also had to lock the variable that keeps track of the size of the list. 

In Software Engineering II we added smart pointers to our script project. Smart pointers are template classes in C++ that automatically deallocate memory when it's no longer needed. When a smart pointer goes out of scope it's destructor is automatically called. This was a crucial part of the project as it eliminated any data leaks that were occurring. After adding the smart pointer header file that declared the new pointers, I used AI to go through my project and replace my old pointers with smart pointers.
March 18-22, 2024This was a busy week. My primary focus was on a project for my Networks and Security class, centering on Transport Layer Security (TLS). Through this, I gained a much better understanding of Diffie-Hellman key generation and cipher block chaining. Additionally, we conducted a lab where we utilized a buffer overflow vulnerability to bypass a password-protected program. Due to a coding error, input beyond the intended password length was processed, allowing for manipulation of the login verification by inserting "junk" data up to the point where the success address on the stack could be altered.

In my Operating Systems class, we starting talking about thread synchronization using locks, transitioning from managing code safety in Java through synchronized sections to applying these concepts in C++. Our upcoming project asks us to develop a thread-safe queue in C++, contrasting its performance and safety with a standard queue, to solidify our understanding of locking mechanisms in managing concurrent processes.

For my Software Engineering II class, we did a quick lab on profiling code to optimize performance, as well as completing another assignment for MSDscript where I implemented function calls. This allows my program to be able to parse and evaluate function like "_let f = _fun (x) x+1 _in f(10)" which would compute to 11. 
March 11-15, 2024My main focus this week was in Operating Systems; my assignment was to create an alternative to the malloc() function in C++, using only mmap (and munmap for deallocation). I designed a system backed by a custom hashmap to track allocated memory addresses and sizes. Upon a call to my malloc replacement, it calculates the minimal pages required for the requested memory, finds an appropriate slot in the hashmap, and logs the allocation details, ensuring efficient and organized memory management.

In Networks and Security I had a checkpoint due for a larger assignment due in two weeks. I generated a bunch of secure keys in the terminal, then explained briefly how each one would be used during runtime when I finish my Transport Layer Security implementation. 

Software Engineering II was heavily project-based this week. I updated my program to work with conditional statements like _if _true _then 4 _else 5, which by itself would simply evaluate to 4, however this update can also be used in _let statements which was pretty tricky to get working. Updating my parsing script was the hardest part of this assignment for me, though. Having to switch through _if, _else, and _then took a lot of new calls.
